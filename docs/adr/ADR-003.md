# ADR-003: Migración de Next.js a Python/FastAPI

**Fecha**: 2026-03-04
**Estado**: Aprobado
**Decisor**: Josep + Atlas (GACE)

## Contexto

DiversIA Eternals v1.0 se construyó como monolito Next.js con Prisma/PostgreSQL. El sistema tiene 272 tests pasando y está listo para producción. Sin embargo, la evolución del producto hacia un matching basado en AI con vectores de 24 dimensiones y modelo trilateral requiere capacidades que Python gestiona mejor.

## Decisión

Migrar el backend completo de Next.js/TypeScript a Python/FastAPI.

## Razones

1. **Ecosistema AI/ML**: Python domina en AI — numpy, scipy, scikit-learn nativos para vectores 24D y cosine similarity
2. **SQLAlchemy 2.0**: ORM async con transacciones explícitas (`async with session.begin()`) — resuelve issue #41
3. **Pydantic v2**: Validación más potente que Zod, integrada nativamente con FastAPI
4. **OpenAPI automático**: FastAPI genera docs Swagger sin configuración — resuelve issue #65
5. **Self-hosted LLM**: Ollama tiene SDK Python oficial, mejor integración
6. **Rendimiento**: FastAPI es comparable a Node.js en benchmarks async
7. **Aprendizaje**: Python/FastAPI es un activo profesional más demandado en el mercado actual

## Alternativas Consideradas

| Alternativa | Pros | Contras | Veredicto |
|------------|------|---------|-----------|
| Mantener Next.js | Sin rewrite, 272 tests | Ecosistema AI limitado, bugs actuales (#38,#39,#40) | Rechazado |
| NestJS (Node.js) | Mismo lenguaje, TypeORM | No resuelve el gap AI/ML, menos natural para vectores | Rechazado |
| Go + FastAPI | Rendimiento extremo | Complejidad, ecosistema AI limitado, curva de aprendizaje | Rechazado |

## Consecuencias

### Positivas
- Matching 24D implementable con herramientas nativas (numpy)
- Transacciones SQLAlchemy resuelven #41
- OpenAPI automático resuelve #65
- Clean Architecture más natural en Python (domain classes puras)
- Jinja2 server-rendering elimina bugs React (#39)

### Negativas
- Rewrite del backend completo (~2-3 semanas)
- Coexistencia temporal Next.js + FastAPI
- 272 tests deben mantenerse hasta completar migración
- Curva de aprendizaje en Python para el equipo

### Riesgos
- **Mitigado**: Coexistencia — tablas Prisma no se tocan, schemas separados
- **Mitigado**: Tests legacy siguen corriendo durante migración
- **Aceptado**: Duplicación temporal de lógica entre stacks

## Métricas de Éxito

- [ ] matching-service calculando matches con 24 dimensiones
- [ ] Tests pytest ≥80% coverage en domain layer
