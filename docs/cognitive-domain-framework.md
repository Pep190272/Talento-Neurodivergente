> **Status:** v1.0 — Scientific Foundation Document
> **Compliance:** EU AI Act Art. 10, 13 (Transparency + Data governance)

## Purpose

This document establishes the clinical evidence base for DiversIA's cognitive
assessment framework. It maps the platform's assessment domains to recognized
clinical standards from three authoritative sources:

1. **DSM-5** (APA, 2013) — Diagnostic and Statistical Manual of Mental Disorders
2. **ICD-11** (WHO, 2019) — International Classification of Diseases
3. **NICE Guidelines** — National Institute for Health and Care Excellence (UK)

| Standard    | Code / Reference | Key Cognitive Domains |

## Assessment Validity Considerations

### Construct Validity

The DiversIA gamified assessment is a **screening and profiling tool**, not a
diagnostic instrument. Key distinctions:

| Clinical Assessment | DiversIA Assessment |
| ------------------- | ------------------- |
| Administered by licensed clinician | Self-administered, gamified |
| Standardized normative data | Platform-relative norms |
| Used for diagnosis | Used for job matching |
| Protected by copyright | Open platform |
| 1-3 hour administration | ~30 minute completion |

### Ecological Validity

Gamified assessments prioritize **ecological validity** (real-world relevance)
over psychometric precision:

- Tasks resemble actual work activities (e.g., pattern finding, information recall)
- Lower test anxiety than formal neuropsychological testing
- Multiple sessions allowed (accommodating fluctuating performance)
- No time pressure penalties (extended time built-in)

## NICE Guideline Compliance

### NG87 — Autism in under 19s (applicable principles for adults)
- **Principle:** Assessment should identify strengths, not just difficulties
- **DiversIA Implementation:** All scores framed as capabilities; strengths-based language

### CG72 — ADHD (updated 2018)
- **Principle:** Assessment across multiple settings and informants
- **DiversIA Implementation:** Multi-modal assessment (games + quiz + self-report)

### General NICE Principles Applied
1. Person-centered approach → Candidate controls data sharing
2. Shared decision-making → Human-in-the-loop for all AI decisions
3. Reasonable adjustments → Gamified format inherently more accessible
4. Informed consent → GDPR Art. 7 compliant consent flow

## Limitations & Disclaimers

1. DiversIA assessments are **not replacements** for clinical neuropsychological evaluation
2. Scores reflect **workplace-relevant cognitive patterns**, not diagnostic criteria
3. The platform **never suggests diagnoses** — users self-declare if they choose
4. All assessment data is **special category data** under GDPR Art. 9
5. Assessment participation is **optional** — candidates can match without it
6. Cultural and linguistic biases in games are monitored through bias audits
